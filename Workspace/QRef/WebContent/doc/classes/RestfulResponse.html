    Class: RestfulResponse

    Defined in: src/serialization/RestfulResponse.coffee

    Overview

    The response object returned for all RESTful requests.
    This response object supports pagination and returning the total available records using the #setTotal method.

    Examples:

    Reporting a Failure 

    response = new RestfulResponse()
response.failure('Some failure reason', 500)

    Returning Data [Single Object] 

    response = new RestfulResponse()
response.addRecord(someObj)
response.setTotal(1)

    Returning Data [Array of Objects] 

    response = new RestfulResponse()
response.addRecords(someArray)
response.setTotal(someCount)

    Reset to Default 

    response = new RestfulResponse()
response.addRecords(someArray)
response.reset()

    Count Objects in Response 

    response = new RestfulResponse()
response.addRecords(someArray)
myCount = response.getLength()

    Property Summary

    (Number) total

    (Required) The total number of records available.

    (Array<Object>) records

    (Required) The records returned from the server.

    (Boolean) success

    (Required) A boolean value indicating whether the request was successful. True if the request succeeded, false otherwise.

    (String) mode

    (Required) The type of request being responded to by the server. Will always be "rest".

    (Mixed) message

    (Required) A string or object containing an error message from a server side failure.

    (Number) errorCode

    (Required) An error code returned from the server indicating failure. Will always be a standard HTTP error code (4xx, 5xx).
