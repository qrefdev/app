    Class: RpcResponse

    Defined in: src/serialization/RpcResponse.coffee

    Overview

    The response object returned for all RPC requests.
    This response object supports returning a single object or other value as the result of an RPC operation.

    Examples:

    Reporting a Failure 

    response = new RpcResponse(null)
response.failure('Some failure reason', 500)

    Returning Data [Single Object] 

    response = new RpcResponse(someObj)

    Return Data [String Value] 

    response = new RpcResponse(someString)

    Reset to Default 

    response = new RpcResponse(someValue)
response.reset()

    Setting a non-fatal message 

    response = new RpcResponse(someValue)
response.setMessage(someMessage)

    Property Summary

    (Boolean) success

    (Required) A boolean value indicating whether the request was successful. True if the request succeeded, false otherwise.

    (String) mode

    (Required) The type of request being responded to by the server. Will always be "rpc".

    (Mixed) message

    (Required) A string or object containing an error message from a server side failure.

    (Number) errorCode

    (Required) An error code returned from the server indicating failure. Will always be a standard HTTP error code (4xx, 5xx).

    (Mixed) returnValue

    (Required) An object, array, or single value returned as the result of the RPC invocation.
